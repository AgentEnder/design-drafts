tinderbox api reference · v0.4

API reference

Every flag, every exit code, every field in the JSON output.

spark

tinderbox spark <name> [--ttl=<days>] [--from=<ref>]

Creates a new branch named spark/<name> at the given ref and records its TTL.

name
Required. Branch slug. Must match [a-z0-9-]+.
--ttl
Days until the branch is eligible for burn. Default 14.
--from
Base ref. Defaults to the current HEAD.

list

tinderbox list [--json] [--expired-only]

Lists known sparks and their remaining TTL.

--json
Emit machine-readable output. See JSON format.
--expired-only
Only show branches whose TTL has elapsed.

burn

tinderbox burn [--remote] [--force] [--dry-run]

Deletes expired sparks. Refuses to delete branches with unpushed commits unless --force is passed.

--remote
Also delete the matching remote branch on origin.
--force
Delete even if the branch has unpushed commits.
--dry-run
Print what would be deleted without doing it.

Exit codes

0
Success.
1
Generic failure (unknown command, bad flag).
2
Git-level failure surfaced from the underlying invocation.
3
burn refused to delete a branch with unpushed commits.

JSON format

Currently unstable. The shape today:

{
  "version": "0.4",
  "branches": [
    {
      "name": "spark/fix-pricing-typo",
      "createdAt": "2026-05-08T09:12:00Z",
      "expiresAt": "2026-05-22T09:12:00Z",
      "remainingDays": 13,
      "expired": false,
      "hasUnpushedCommits": false
    }
  ]
}
