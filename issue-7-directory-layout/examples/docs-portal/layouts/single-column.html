tinderbox layout: single-column

Single-column reading layout

One column, capped at 68ch, centered on the page. No left TOC, no right outline. The page itself is the navigation: scroll, link, type-ahead-search-the-page, done.

This layout is best for narrative documentation — concept pages, design rationale, anything that benefits from being read in order. It is worse for reference material, where readers want to scan a list of available functions and click on one.

Trade-offs

You give up:

  • Persistent visibility of the doc tree. Readers have to scroll back to the top to navigate elsewhere.
  • The right-rail outline that lets readers jump to a section heading.

You gain:

  • Optimal line length on every screen, automatically.
  • Less chrome, more content.
  • A layout that prints well.
