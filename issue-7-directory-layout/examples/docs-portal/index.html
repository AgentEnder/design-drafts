tinderbox v0.4 · CLI for experiment branches

A small CLI for managing experiment branches

Tinderbox is a single binary you drop into a repo to keep short-lived branches off your main tree. It has three commands: spark, list, and burn. That is the whole tool.

Read the quickstart to install and create your first branch. Skip to the API reference if you already know what you want.

Why it exists

Most teams accumulate a long tail of wip/* branches that nobody owns and nobody deletes. Tinderbox treats those branches as a queue: each spark gets a TTL, and burn reaps anything past it. The default TTL is two weeks.

Status

Tinderbox is pre-1.0. The CLI surface is stable for the three commands above; the JSON output format is not yet frozen. See the API reference for the current contract.
